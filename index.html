Github issues and pull request tutorial

What is Github?

GitHub is a web-based Git or version control repository and Internet hosting service. To understand GitHub, you must first have an understanding of Git. Git is an open-source version control system that was started by Linus Trovalds – the same person who created Linux.

So, Git is a “version control system,” what’s that mean? When developers are creating something (an application, for example), they are making constant changes to the code and releasing new versions, up to and after the first official (non-beta) release. Version control systems keep these revisions straight, and store the modifications in a central repository.

More on Github and how to create a simple repository, you can find here.

What are issues and what will we do in this tutorial

Issues are a great way to keep track of tasks, enhancements, and bugs for projects. They’re kind of like email—except they can be shared and discussed with the rest of the team. Most software projects have a bug tracker of some kind. GitHub’s tracker is called Issues, and has its own section in every repository.

We are going to use issues for keeping track of tasks in this tutorial and in out future projects. Our end product of this tutorial will be this page, that we will assemble as a team.

Github setup

Register & login

First of all, if you don't have an account you should register here.

Once you're done registering or if you already have an account, login.

Fork repository

Now, you'll need to fork this repository.

What is a repository? A repository is usually used to organize a single project. Repositories can contain folders and files, images, videos, spreadsheets, and data sets – anything your project needs.

What does it mean to fork a repository? Creating a “fork” is producing a personal copy of someone else’s project. Forks act as a sort of bridge between the original repository and your personal copy. You can submit Pull Requests to help make other people’s projects better by offering your changes up to the original project. Forking is at the core of social coding at GitHub.

To fork the repository, click the Fork button in the header of the repository.

Setup git on your machine & and clone the repo

Mac & Linux

To install git and clone the repository we'll use the terminal, so open that up.

Install git

If you don't already have git installed you'll need to download and run this installer if you're using macOS and if you're using Linux enter this command on the terminal:
$ apt-get install git

To check if you have git on your machine enter this command:
git --version

Clone repository

Using commands ls(to find out what directory you're currently in) and cd(to travel from one directory to another) navigate to a directory you wish to have this project in.

Open your forked repository in the browser, click on the "Clone or download" button and copy the address of your repository.

Then in your terminal enter the following command, but instead of url use the address of your repository you have just copied
$ git clone url

And now you should have the repository sitting locally on your machine.

Windows

Install git

To have git on Windows, you'll need to download and run this installer. It installs Git GUI, from which you can create, clone and manage your repositories, and Git Bash which you can use for the same purposes just using git commands. In this tutorial we're going to use both Git GUI and Git Bash.

Clone repository

Open your forked repository in the browser, click on the "Clone or download" button and copy the address of your repository.

Once you have installed git on your windows machine and copied the address of your repository, open Git GUI and click "Clone existing repository". Then paste your repository address into source location field and select your target directory as the directory you want your project to be in.

Now all your repository files should be on your machine

Set up your user profile

To finish setting up git on your machine you have to set up your user profile. In order to do that you have to open your terminal (mac & linux) or Git Bash (windows) and enter these two commands just replace the email and name with your details:
$ git config --global user.email "you@example.com"
$ git config --global user.name "Your Name"

Code writing

Install Atom

For editting and commiting our code we'll be using a text editor called "Atom". If you don't have it yet, download and install it from here.

Once you have successfully installed it we'll need to install a git package that will help us easily push our changes from the text editor.
So in the top menu select File -> Settings -> Install, search for "git-plus" and install the package.

Open cloned project

In the top menu select File -> Open Folder... and choose the folder that your cloned project lays in.

Assigned issue

Open up the original repository in your browser and click on the issues section. By now you should have been assigned an issue, if not then scream like a hungry baby bird, maybe someone will pay attention to you.

Open up the issue that has been assigned to you and follow the instrctions on what you should do with your code. Here is where the scary word "team" comes in again, since we all have different tasks and we need to finish them all in order for the page to be finished. So come back to atom and make all your changes there.

Add, Commit & Push

After you're done with your task, right click on your project (on your left) and select Git -> git add + commit. Then enter your commit message, saying what changes you have done and save it.

When that is successfully done right click on your project again and this time select Git -> git push.

Pull request

You have pushed your changes to your forked repository, but not to the original one. To do that you'll need to create a pull request.

So go back to the original repository in your browser and click on the pull requests section.

Step Screenshot
From the Pull Request page, click the green New pull request button.
Look over your changes in the diffs on the Compare page, make sure they’re what you want to submit.
When you’re satisfied that these are the changes you want to submit, click the big green Create Pull Request button.
Give your pull request a title and write a brief description of your changes. In your description you should include a word "close" and the issue number that was assigned to you, which will automatically close your issue after your changes have been merged with the master branch. More info about that.

Final result

Now you'll have to wait until the creator of the original repository will decide to merge your request. And after all requests are merged you should be able to see the final result here!
